Chapter 1: - Introduction to the topic

1.1 - Problem Domain

Global Positioning system (GPS) is able to accurately determine the outdoor location of objects or people by using signals from many satellites at the same time to calculate the distance of the object or people with a technique known as trilateration[1]. But it becomes inaccurate when trying to use it to determine the position of a person or an object within a building or any indoor location. The signals provided by the GPS satellites are easily degraded once there is no direct line-of-sight with the satellites from the object connecting to it. Also, GPS cannot determine the floor in a building a person or an object is. Since the information provided the GPS is not accurate, there is a need to find suitable approach that can provide significantly better result.

The need to determine the location and position of objects, people in a building has become more important in recent times for businesses such as shopping-malls, hotels, museums, airports, universities, inventory and logistics companies who need to be able to track and monitor movement in and out of their facilities and for personal use in a smart home environment where a user needs to be identified in order to personalize the service offered by smart devices. With tremendous improvement being achieved in the area of wireless connectivity, coupled with the development of IoT devices with various sensors which has contributed to the rise in popularity of smart homes[2].

1.2 - Introduction to Indoor User Location Identification in a smart home

First of all, it is ideal to understand what a smart home is and one good definition says that “A smart home, or smart house, is a home that incorporates advanced automation systems to provide the inhabitants with sophisticated monitoring and control over the building's functions. For example a smart home may control lighting, temperature, multimedia, security, window and door operations, as well as many other functions”[3].

Year by year, the area of building smart home is enlarging and having more capabilities, features. One of the new features is the voice controlling where a user should be able to command by the voice and all functions which are mentioned in core definition should be implemented by this voice command.

1.3 - Our research direction

Voice command for the smart home is a new trend and smart home system need to receive user commands via voice. At the same time, smart home system should be aware of the location of the user while receiving commands, in order to provide better service. For example, if a user walks to Kitchen at night and says “Switch the light on”, then the system should switch the light on up to the user reaches to the Kitchen. The user does not need to mention which light or light, there might be multiple lights until Kitchen such corridors, stairs. There have been many ways to know approximate location of the user such as using Beacons, Bluetooth or WiFi technologies. But all of those approaches requires the user to carry a device.
Therefore, we have decided to make research on the voice location for indoor user location identification for a smart home.

4.3 - Discovering capabilities of Matrix Voice

It is obvious that we need microphones which can take clear voice from long distance even if the voice level is really low. In order to know the reliability of clear voice delivery from Matrix Voice, we have decided to integrate Matrix Voice with Google Assistant and test the distance by speaking from different distances. In other words, we have customized Google Assistant which uses Matrix Voice for voice input and we have spoken to that Customized Google Assistant from different range of distance such as starting 1 meter up to 15 meters and from collected data, we could conclude that the voice between 1 and 12 meters is fully delivered and from 12 meters the voice’s quality is low and Google Assistant was not recording correct context.

4.4 - Determining the direction of the voice

The direction of the user’s voice is possible to determine with Matrix Voice due to the fact that Matrix Voice has microphones which are collected in certain cardinalities and from those cardinalies can determine the direction of the voice. As Figure 4.2, shows that each microphone has X and Y cardinalities and each microphone produces distinct voice strength, depending on which a microphone is targeted.

As long as the voice is recorded, each microphone produces different voice strength as it is shown in Figure 4.2. For example, as in Figure 4.3, the user speaks towards microphones and we can check a voice strength and compare it with actual voice strength of each microphone. In the Figure 4.4, we have described that a user speaking to Microphone M3 and it has voice strengths between 12 000 and 16 000, and we compare a taken voice strength in that range as it is shown. If the received voice strength is between those range, we can say that a user is speaking to Microphone M3 and most importantly from this point, we can determine the direction due to cardinalities of M3.
Having understood working principles of obtaining a direction of the voice, we can conclude that this approach can be enhanced by increasing the number of Matrix Voices devices (input points). To get optimal a direction of the voice, it is enough to have 3 Matrix Voices input points. The Figure 4.5 shows that we can obtain an intersection point of all cardinalities of 3 Matrix Voice while those 3 Matrix Voice will receive the voice and produce different cardinalities for a targeted microphone.
